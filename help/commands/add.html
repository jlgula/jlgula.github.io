add

NAME

add - adds new devices to the grid

SYNOPSIS

add type [name]

DESCRIPTION

Adds a new device to the simulated bus. By default, the new device is assigned a name corresponding to its type with a number appended to make the name unique on the current bus. The device is assigned a URL with the scheme "sim:" and authority corresponding to its name.

The default name can be overriden with an explicit name which is also used for its URL.

The type must be one of the supported device types as listed in the devices command. An unrecognized type will result in a "Not found" error. Note that the types are case sensitive.

EXAMPLES

add Light

Adds a new Light device to the bus using the name "Light1" assuming there are no other lights on the bus.

add LPDGridController gc

Adds a new LPDGridController to the bus and gives it the name "gc" with URL: sim://gc

NOTES

The browser version of the simulator only supports devices using the scheme "sim:". The JVM desktop version supports regular http devices that listens on a particular port. A separate thread is launched to control the device, thus one can do something like:

add Light:8080

which creates a thread listening to port 8080 for regular http messages and one can point a browser at it with something like:

chrome http://localhost:8080.
