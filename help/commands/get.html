NAME

get - displays the value of the ThingSet data element at the path

SYNOPSIS

get path

DESCRIPTION

Sends the Get HTTP/ThingSet message to the device as a request for a ThingSet value. The value is represented and displayed in JSON. The path can be a relative URL starting with the device name or an absolute URL starting with the scheme (always "sim:" in the web app).

If the path ends with a "/" or the path is just the device name or URL, the message is interpreted as a request to list the property names available at the path and can thus be used to explore the tree of values available in ThingSet via the NameID command. The result is a JSON array of the names as text strings. For arrays, if the path to the array ends in slash, the result is an array of the index values of the array. These are always strings that look like numbers but are just unique values rather than having any relationship to an array index.

The path can be shortened or elided completely using the attach command.

EXAMPLES

>> get Light1/on

true

>> get sim://Light1/on

true
>> get Light1/simulatedAttributes/

["simulator","time","deviceCurrent","vBus","circuitElements"]
>> attach Light1

Attached: sim://Light1

>> get

["loadResistance","on","suspended","price","priceLimit","gridController","subscription","simulatedAttributes"]
>>add ChargeController
AddCommand: device ChargeController1 added at sim://ChargeController1

>>post ChargeController1/test/arrayi32 7
Created: /test/arrayi32/0

>>get ChargeController1/test/arrayi32/
["0"]
