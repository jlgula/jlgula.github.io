Commands

The bottom portion of the application contains a field for entering commands via a command line and displays the results of excuting the command below. The Run button isn't needed; the command will excute upon pressing the return key on keyboard but the button can also be used.

The following is a brief overview of the commands. Click the link for more information. Some of the commands have more functionality in the desktop JVM version of the application that cannot be duplicated in the browser platform.

CommandArguments         Description
addtype [name]Adds a new device to the grid. The type argument must be a supported type as shown by devices.
attach[name] | [URL]Sets the start of the path for commands like get that include a path to shorten those commands.
circuitDisplays the complete circuit analysis of nodes and currents.
deletepathSends an HTTP/Thingset delete message to the device to delete an array element at the path which must include its index.
devicesDisplays a list of the types of devices that can be added to the grid.
exitCurrently ignored in the web app.
getpathSends an HTTP/ThingSet get message and displays the value of the ThingSet data element at the path as a JSON value.
help[command]Displays a list of the commands or more detail on a specific command.
loadLoads a file from the file system.
logDisplays the log of messages exchanged between devices.
patchpath valueSends an HTTP/ThingSet patch message. The value must be a JSON object.
postpath valueSends an HTTP post message (ThingSet create). Sets an array value and returns the index of the value.
putpath valueSends an HTTP put message (not used in ThingSet). Sets a value encoded in JSON.
remove[name] | [URL]Removes a device from the grid.
scheduledelay path valueSchedules a put message to be delivered to the path at a time offset from the start of the simulation.
schematypeDisplays the full schema of a device type.
simulate[count [interval]]Runs a simulation of count (default 1) ticks and displays the results. The interval is the delay between ticks in seconds - defaults to 1 second.
version[-V]Shows the version of the application.
